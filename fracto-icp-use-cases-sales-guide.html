Sales Intern Glossary
Chapter 1 - Glossary

Explain these words once, then use simpler language on calls.

OCR
Reads text from an image or PDF. It does not automatically know what the business should do next.
API
A way for two software systems to talk to each other automatically.
Webhook
Fracto sends the result automatically to another system when processing is done.
Review queue
A list of files that still need a human to check because something is unclear or missing.
Confidence
How sure the system is about an extracted answer or decision.
Master data
The company's official list, like SKU list, vendor list, customer list, or product list.
Validation
Checking whether extracted data is complete, correct, and follows the company's rules.
Exception
A case that cannot be passed automatically and needs human review.
Integration
Connecting Fracto output to tools like email, Google Sheets, ERP, CRM, or internal software.
Intern rule: if the lead is non-technical, say "send result to your system" instead of "API/webhook integration."

Product Mental Model
Chapter 2 - Product Basics

Fracto is workflow automation around messy documents, not just OCR.

What Fracto is
Ops-ready document AI

It reads messy files, applies business checks, routes exceptions, and sends clean output to systems.

What it is not
Not generic OCR

It is not a one-off extraction tool, template-only parser, BPO team, or generic LLM prompt.

Core buyer pain
Manual review load

Teams open files, check rules, type into systems, and chase missing or unclear cases.

Sales anchor
File in, decision out

Every pitch should map input, checks, review path, output, owner, and success metric.

1
IngestionEmail, WhatsApp, app, API, SFTP, sheets, folders, PDFs, and images.
2
ExtractionFields, tables, line items, handwriting, stamps, signatures, IDs, and remarks.
3
ValidationRules, completeness, confidence, duplicate checks, masters, and verification APIs.
4
ReviewOnly unclear, missing, invalid, or low-confidence cases go to humans.
5
IntegrationsJSON, CSV, webhook, ERP, LOS, TMS, CRM, Google Sheets, and dashboards.
6
AnalyticsVolume, auto-pass, exception reasons, SLA, reviewer load, and accuracy reporting.
7
ControlsAccess, audit trail, masking, retention, and customer-approved handling.
8
ExpansionOnce one workflow works, add adjacent documents and checks.
Mental model for reps: OCR reads text. Document AI understands fields. Workflow automation moves decisions. Fracto combines these around the customer's operational process.

Buyer Language
Chapter 3 - Listen For

The buyer will usually describe work, not "AI."

Manual review
"Our ops team checks every file."
Retyping
"Data goes from WhatsApp or PDF into ERP."
Late exceptions
"Missing stamps or wrong files are found too late."
SLA pressure
"Volume spikes create backlogs."
Matching work
"We compare invoice, PO, GRN, and vendor master."
Evidence sorting
"Claims or returns need photos, bills, and forms checked."
Translate to Fracto
  • Manual checking -> extraction + validation rules
  • Retyping -> structured API / CSV / ERP payload
  • Late misses -> completeness and exception flags
  • Document matching -> compare fields across files and masters
  • Backlogs -> auto-pass clean cases, review only exceptions
Avoid this pitch

"We do AI OCR." It is too generic. Sell the workflow outcome: faster proofing, cleaner onboarding, quote-ready rows, or underwriting-ready packets.

ROI and Competition
Chapter 4 - ROI and Alternatives

Make cost visible during discovery.

Formula
  • Monthly volume x minutes per doc = manual minutes
  • Rs 25,000 / 26 / 8 = about Rs 120/hour
  • Manual minutes / 60 x Rs 120 = base labor cost
  • Add rework, delay, SLA penalty, leakage, and blocked revenue
  • Compare against Fracto subscription + credit overage
Example: 10,000 docs x 2 min = 20,000 min = 333 hours. At Rs 120/hour, base review cost is about Rs 40,000/month before delay and rework.
Competitive Positioning

Chapter 6 - Pricing Guardrails

Keep pricing consistent and avoid early commitments.

Model
Subscription + credits

Monthly subscription includes page credits; API verification credits are separate add-ons.

Pilot
Paid + refundable

Refundable if it does not work within first 30 days against agreed success criteria.

Credit unit
Page by default

Escalate
Custom pricing

Escalate high volume, strict SLA, on-prem, complex security, or heavy custom integration.

Discounts
Do not improvise

Discounts need approval and should be tied to volume, term, or strategic value.

Do not promise
No 100% accuracy

Promise measured pilot results, review routing, and agreed success metrics.

Quote after demo
Scope first

Need volume, page count, checks, APIs, SLA, and integration path before final quote.

Minimums
Use current policy

Confirm minimum pilot and production contract values internally before quoting.

Annexure: FAQ A
Annexure - Commercial FAQ

Use these answers for pricing and pilot conversations.

How is Fracto priced?
As a monthly subscription with included credits. Usage beyond included credits is billed per additional credit.
What is a credit?
For document workflows, one credit usually maps to a page. Verification APIs are counted per verification call.
What inputs are needed for a quote?
Monthly cases, pages/images per case, required fields, validation rules, SLA, integrations, and expected review process.
Do we need to share samples before the demo?
No. First qualify and book the demo. Samples are best requested after the buyer agrees the workflow and pilot shape.
Is there a pilot or proof of concept?
Yes. The pilot is paid and refundable if it does not work for the client within the first 30 days.
How do we justify ROI?
Calculate manual effort using Rs 25,000/month as base salary. Estimate minutes per page/doc, monthly volume, and reviewer count.
What if usage crosses included credits?
The subscription includes a credit bundle. Extra usage is billed per additional credit at the agreed overage rate.
Can the plan change after volume grows?
Yes. Start with expected monthly volume, then move to a higher credit bundle when usage becomes predictable.
How should we position the paid pilot?
It protects both sides: the client gets a real test, and the fee is refundable if it does not work within 30 days.
ROI shortcut: Rs 25,000 / 26 days / 8 hours is roughly Rs 2 per working minute. If 10,000 docs take 2 minutes each, base manual cost is about Rs 40,000/month before rework and delays.

Annexure: FAQ B
Annexure - Product FAQ

Explain how Fracto fits into real operations workflows.

Can Fracto run sync and async?
Yes. Sync works for real-time API/app decisions. Async works for batches, queues, email, WhatsApp, or large PDFs.
When should we pitch sync?
Use sync when the user is waiting inside an app or portal and needs a quick accept, reject, or review response.
When should we pitch async?
Use async when files arrive in bulk and the team can process them in minutes, batches, or review queues.
What outputs can Fracto return?
Structured JSON, CSV, webhooks, dashboard queues, ERP/LOS/TMS payloads, confidence scores, and review flags.
Does it work with product masters?
Yes. It can match aliases to SKUs, vendor masters, customer masters, policy rules, or document checklists.
What about handwriting or poor image quality?
Fracto can handle messy inputs, but unclear cases should be flagged to review. Do not promise 100% automation.
Can it extract tables and line items?
Yes. Good fits include orders, invoices, packing lists, loan packets, and shipment documents with repeated rows.
Can it validate against business rules?
Yes. Rules can check required fields, ranges, master matches, document completeness, and review triggers.
Can it process images and PDFs together?
Yes. Fracto can classify mixed inputs first, then route each image, PDF, or page to the right extraction flow.
Best answer for accuracy: "We measure field accuracy, decision accuracy, and auto-pass coverage. Low-confidence cases go to review."

Annexure: FAQ C
Annexure - Implementation FAQ

Use these answers for security, integrations, and rollout concerns.

How does Fracto integrate?
Common paths: email, WhatsApp, Google Sheets, Excel/CSV, API, webhook, SFTP, dashboard queue, or ERP/LOS/TMS mapping.
Where does human review happen?
Clean cases can auto-pass. Low-confidence, missing, invalid, or rule-failed cases go to a review queue with reasons.
How long does rollout take?
If not highly custom, Fracto can give a testing flow in 1 to 2 days. After testing, integration is usually one sprint or 1 to 2 mandays.
Is customer data secure?
Position security as part of implementation: secure transfer, access control, retention policy, audit needs, and environment separation.
Can Fracto handle PII?
Yes, with the right workflow controls: masking, redaction, role-based access, retention limits, and customer-approved handling.
What if the process changes?
Rules, fields, product masters, and exception logic can be updated. Confirm ownership for change requests during rollout.
Who needs to join the demo?
Bring the ops owner, the person who reviews exceptions, and the system owner for ERP, LOS, TMS, or internal tools.
What if the client has no API?
Start with email, Google Sheets, CSV, dashboard, or SFTP. API/webhook can come later after value is proven.
Can rollout happen branch by branch?
Yes. Start with one workflow, branch, region, or team, then expand once rules and review flow are stable.
Safe close: "Let us map this in the demo with your ops and system owner, then we can confirm pilot scope, data handling, and integration path."

Annexure: FAQ E
Annexure - Objections FAQ

Use these when the buyer pushes back during discovery.

"We already have OCR."
Good. Fracto sits after OCR: classification, validation, master matching, confidence, and exception routing.
"Can our tech team build this?"
Maybe, but Fracto is faster when the pain is operational and the team needs extraction, rules, review, and iteration.
"Our formats keep changing."
That is normal. Sell review flags and configurable fields/rules, not a brittle fixed-template parser.
"We do not want full automation."
That is fine. Start with auto-pass for clean cases and human review for unclear or high-risk cases.
"Volume is not very high."
Check if the delay is high-value: billing blockage, onboarding loss, compliance risk, or senior-team review time.
"Accuracy must be 100%."
Do not promise that. Say clean cases can auto-pass and low-confidence cases go to review with reasons.
"This sounds expensive."
Bring it back to volume and manual cost. Compare subscription cost with reviewer time, delays, and rework.
"We need management approval."
Offer a demo with the ops owner and decision maker. Use the pilot to create proof, not a long proposal first.
"We cannot share real data."
Start with redacted samples or synthetic examples for demo. For pilot, align on secure handling and retention.
Strong response: "Fracto is not replacing every reviewer on day one. It removes repetitive checks and makes review queues smaller and cleaner."

Annexure: FAQ F
Annexure - Edge Cases

Use these answers when the buyer asks about messy real-world files.

What if a document is duplicate?
Duplicates can be flagged using document fields, file metadata, shipment/order IDs, or similarity checks.
What if required fields are missing?
Fracto returns missing-field flags and sends the case to review instead of forcing a false clean output.
What if files are mixed together?
Fracto can classify pages or attachments first, then route each document type to the right extraction flow.
Can it handle multiple languages?
Often yes, depending on the fields and scripts. Ask for sample variety after demo to confirm coverage.
What if master data is messy?
Start with the best available master and alias list. Unknown or ambiguous matches should go to review.
Can outputs be audited?
Yes. Keep source file, extracted fields, confidence, decision, reason codes, and reviewer action where needed.
What if the wrong document is uploaded?
Fracto can classify the file and flag wrong-document cases before they enter the main workflow.
What if a PDF has 100+ pages?
Use async processing. Fracto can split, classify, and process pages, then return a structured packet summary.
What happens to low-confidence fields?
They should be returned with confidence and reason codes, then routed to review or customer correction.
Discovery prompt: "Show us the top 5 messy cases your team hates. That usually tells us the real exception rules."
